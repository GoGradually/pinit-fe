## 사용 방법

> 필요에 따라 파일을 추가하거나 이름을 바꿔도 됩니다. 다만, `core-system-template.md`를 베이스로 삼는 패턴은 유지하는 것을 권장합니다.

  모든 에이전트가 공유하는 응답 스타일 & 베스트 프랙티스.
- `agent-style-guide.md` (선택)  
  데이터 분석 / 리포트 전용 에이전트 템플릿.
- `agent-data-analyst.md`  
  제품 기획 / UX 카피 전용 에이전트 템플릿.
- `agent-product-ux.md`  
  버그 헌팅/디버깅 전용 에이전트 템플릿.
- `agent-bug-hunter.md`  
  코드 리뷰 전용 에이전트 템플릿.
- `agent-code-review.md`  
  공통 시스템 프롬프트 템플릿. 모든 에이전트의 베이스가 되는 구조와 섹션을 정의합니다.
- `core-system-template.md`  

## 디렉터리 구조

이 디렉터리는 팀에서 사용하는 LLM 에이전트(Agent)들의 **시스템 프롬프트 템플릿**을 모아둔 곳입니다. 새 에이전트를 만들거나, 기존 에이전트의 역할/스타일을 정교하게 다듬고 싶을 때 이 파일들을 참고하거나 복사해서 사용하세요.
